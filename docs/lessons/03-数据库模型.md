### 这是开发顺序的第一步：Model层

```
Model → DAO → Service → Controller → 路由注册

① Model层: ★当前步骤★ 定义数据结构（用户、群组、会话、消息等表）

```

## 一、GORM简介

### 什么是GORM？

GORM是Go语言最流行的ORM（对象关系映射）库：
- 用Go结构体定义数据库表结构
- 自动创建/更新表结构（AutoMigrate）
- 提供丰富的查询API

### 安装GORM

```bash
go get gorm.io/gorm
go get gorm.io/driver/mysql
```

## 二、UserInfo 用户表

**文件位置:** `internal/model/user_info.go`

根据文档 `4.后端开发.md` 行26-41：

```go
package model

import (
    "database/sql"  // ★新增：用于sql.NullTime类型
    "time"
    
    "gorm.io/gorm"
)

// UserInfo - 用户表模型
// ★所有字段都使用 column:xxx 标签指定数据库列名（蛇形命名）
type UserInfo struct {
    Id            int64          `gorm:"column:id;primaryKey;comment:自增id"`
    
    // ★Uuid是唯一索引，格式: "U" + 11位随机字符
    Uuid          string         `gorm:"column:uuid;uniqueIndex;type:char(20);comment:用户唯一id"`
    
    Nickname      string         `gorm:"column:nickname;type:varchar(20);not null;comment:昵称"`
    
    // ★普通索引而非唯一索引！因为软删除后可以重新注册
    Telephone     string         `gorm:"column:telephone;index;not null;type:char(11);comment:电话"`
    
    Email         string         `gorm:"column:email;type:char(30);comment:邮箱"`
    
    // 默认头像URL
    Avatar        string         `gorm:"column:avatar;type:char(255);default:https://cube.elemecdn.com/0/88/03b0d39583f48206768a7534e55bcpng.png;not null;comment:头像"`
    
    // 0=男, 1=女
    Gender        int8           `gorm:"column:gender;comment:性别，0.男，1.女"`
    
    Signature     string         `gorm:"column:signature;type:varchar(100);comment:个性签名"`
    
    Password      string         `gorm:"column:password;type:char(18);not null;comment:密码"`
    
    // 格式: YYYYMMDD，如 "20060116"
    Birthday      string         `gorm:"column:birthday;type:char(8);comment:生日"`
    
    // ★蛇形列名: created_at
    CreatedAt     time.Time      `gorm:"column:created_at;index;type:datetime;not null;comment:创建时间"`
    
    // ★软删除字段 - GORM查询时自动过滤已删除数据
    DeletedAt     gorm.DeletedAt `gorm:"column:deleted_at;type:datetime;comment:删除时间"`
    
    // ★新增：用户在线状态相关字段
    LastOnlineAt  sql.NullTime   `gorm:"column:last_online_at;type:datetime;comment:上次登录时间"`
    LastOfflineAt sql.NullTime   `gorm:"column:last_offline_at;type:datetime;comment:最近离线时间"`
    
    // 0=普通用户, 1=管理员
    IsAdmin       int8           `gorm:"column:is_admin;not null;comment:是否是管理员，0.不是，1.是"`
    
    // ★Status添加index，便于快速查询禁用用户
    // 0=正常, 1=禁用
    Status        int8           `gorm:"column:status;index;not null;comment:状态，0.正常，1.禁用"`
}

// ============================================================
// ★新增：TableName方法 - 指定表名
// ============================================================
func (UserInfo) TableName() string {
    return "user_info"
}
```

### UserInfo 设计要点

**1. Uuid为什么用唯一索引？**
- 用于唯一识别用户
- 格式: "U" + 11位随机字符
- 不用自增Id的原因：Id会暴露用户数量，不安全

**2. Telephone为什么用普通索引？**

文档原文（行46-47）：
> Telephone是普通索引，之所以不是唯一索引是因为软删除可能会删除某个Telephone，但该表项还保留在数据表中，再次用该Telephone新建时如果是唯一索引，就无法新建

```go
// 如果用唯一索引（错误）
Telephone string `gorm:"uniqueIndex"`  // ❌ 软删除后无法重新注册

// 正确做法：用普通索引
Telephone string `gorm:"index"`         // ✓ 软删除后可以重新注册
```

**3. 为什么用 `column:xxx` 标签？**

```go
CreatedAt time.Time `gorm:"column:created_at"`
```

| 不写column | 写column | 结果 |
|-----------|---------|------|
| GORM用驼峰命名 | 强制用蛇形命名 | 数据库列名是 `created_at` |

```go
package model

import (
    "encoding/json"
    "time"
    
    "gorm.io/gorm"
)

type GroupInfo struct {
    Id        int64           `gorm:"column:id;primaryKey;comment:自增id"`
    Uuid      string          `gorm:"column:uuid;uniqueIndex;type:char(20);not null;comment:群组唯一id"`
    Name      string          `gorm:"column:name;type:varchar(20);not null;comment:群名称"`
    Notice    string          `gorm:"column:notice;type:varchar(500);comment:群公告"`
    
    // ★json.RawMessage：GORM不支持[]string，用JSON存储
    Members   json.RawMessage `gorm:"column:members;type:json;comment:群组成员"`
    
    MemberCnt int             `gorm:"column:member_cnt;default:1;comment:群人数"`
    OwnerId   string          `gorm:"column:owner_id;type:char(20);not null;comment:群主uuid"`
    AddMode   int8            `gorm:"column:add_mode;default:0;comment:加群方式，0.直接，1.审核"`
    Avatar    string          `gorm:"column:avatar;type:char(255);default:https://cube.elemecdn.com/0/88/03b0d39583f48206768a7534e55bcpng.png;not null;comment:头像"`
    Status    int8            `gorm:"column:status;default:0;comment:状态，0.正常，1.禁用，2.解散"`
    CreatedAt time.Time       `gorm:"column:created_at;index;type:datetime;not null;comment:创建时间"`
    // ★新增：群组更新时间
    UpdatedAt time.Time       `gorm:"column:updated_at;type:datetime;not null;comment:更新时间"`
    DeletedAt gorm.DeletedAt  `gorm:"column:deleted_at;index;comment:删除时间"`
}

// ============================================================
// ★新增：TableName方法
// ============================================================
func (GroupInfo) TableName() string {
    return "group_info"
}
```

**设计要点：Members用json.RawMessage**

文档原文（行84-85）：
> Members是按理是一个[]string类型的数组，其中存储着每个群成员的uuid。但是gorm不支持直接存入[]string，所以使用到了json.RawMessage

```go
// 存储时：序列化
var members []string = []string{"U123", "U456"}
group.Members, _ = json.Marshal(members)

// 取出时：反序列化
var members []string
json.Unmarshal(group.Members, &members)
```

## 四、GORM标签详解

| 标签 | 作用 | 示例 |
|------|------|------|
| `primaryKey` | 主键 | `Id int64` |
| `uniqueIndex` | 唯一索引（不能重复） | `Uuid` 不能重复 |
| `index` | 普通索引（可重复，加速查询） | `Telephone` 可重复 |
| `type:char(20)` | 固定长度字符串 | 固定20字符 |
| `type:varchar(20)` | 可变长度字符串 | 最长20字符 |
| `type:json` | JSON类型 | 存储数组 |
| `type:TEXT` | 大文本 | 消息内容 |
| `type:datetime` | 时间类型 | 创建时间 |
| `not null` | 不能为空 | 必须有值 |
| `default:xxx` | 默认值 | 不设置时用默认 |
| `comment:xxx` | 字段注释 | 写入数据库的注释 |
| `column:xxx` | 指定数据库列名 | 强制用蛇形命名 |

## 五、软删除详解

### gorm.DeletedAt 的原理

文档原文（行50-64）：

```go
// gorm.DeletedAt 内部结构
type DeletedAt sql.NullTime

// sql.NullTime 结构
type NullTime struct {
    Time  time.Time  // 删除时间
    Valid bool       // true=已删除, false=未删除
}
```

### 软删除的行为

```
未删除时：
- DeletedAt.Valid = false
- deleted_at 字段为 NULL
- GORM查询可以找到这条记录

已删除时：
- DeletedAt.Valid = true
- deleted_at 字段有值（删除时间）
- GORM查询自动过滤这条记录
```

### GORM查询自动加条件

```go
// 查询所有用户
dao.GormDB.Find(&users)

// GORM自动生成的SQL:
// SELECT * FROM user_info WHERE deleted_at IS NULL
```

### 如何查询包含已删除的记录？

```go
// 使用 Unscoped()
dao.GormDB.Unscoped().Find(&users)

// 生成的SQL:
// SELECT * FROM user_info  (没有deleted_at条件)
```
