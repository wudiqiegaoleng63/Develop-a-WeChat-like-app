## 五、两种模式对比

### Channel模式

```
┌────────┐   Read()   ┌────────────┐   SendMessageToTransmit()   ┌────────────┐
│  前端   │ ───────── │   Client   │ ───────────────────────── │   Transmit  │
└────────┘            └────────────┘                           └──────┬─────┘
                                                                   │
                                                                   │ case data
                                                                   ▼
┌────────┐  Write()  ┌────────────┐   SendBack Channel    ┌──────────────┐
│  前端   │ ◄──────── │   Client   │ ◄─────────────────── │ Server.Start │
└────────┘            └────────────┘                      └──────────────┘

特点：
- 内存Channel传递
- 容量有限 (CHANNEL_SIZE = 100)
- 高峰期可能阻塞
- 适合小规模 (< 100用户)
```

### Kafka模式

```
┌────────┐   Read()   ┌────────────┐   WriteMessages()   ┌────────────────┐
│  前端   │ ───────── │   Client   │ ───────────────── │  Kafka Topic    │
└────────┘            └────────────┘                    │  (chat_message) │
                                                        └───────┬────────┘
                                                                │
                                                                │ ReadMessage()
                                                                ▼
┌────────┐  Write()  ┌────────────┐   SendBack Channel    ┌──────────────┐
│  前端   │ ◄──────── │   Client   │ ◄─────────────────── │KafkaChatServer│
└────────┘            └────────────┘                      │   .Start()    │
                                                        └──────────────┘

特点：
- Kafka消息队列传递
- 容量无限（磁盘存储）
- 不阻塞，支持持久化
- 适合大规模 (> 1000用户)
- 支持分布式部署
```

## 六、消息流转完整流程

### 私聊消息流程

```
用户A发送消息
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. ClientA.Read()                                           │
│    - 从WebSocket读取消息                                     │
│    - 根据messageMode选择中转方式                             │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 消息中转                                                  │
│    Channel模式: ChatServer.Transmit                         │
│    Kafka模式:  Kafka Topic (chat_message)                   │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. Server.Start() 消费消息                                  │
│    - 解析 ChatMessageRequest                                │
│    - 存储到数据库 (Message表)                               │
│    - 判断 receiveId[0] == 'U' (私聊)                       │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 转发消息                                                  │
│    - 查找用户B是否在线 (Clients map)                        │
│    - 在线: ClientB.SendBack <- messageBack                 │
│    - 回显: ClientA.SendBack <- messageBack                 │
│    - 更新Redis缓存                                          │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. Client.Write()                                           │
│    - 从SendBack读取消息                                     │
│    - 通过WebSocket发送给前端                                │
│    - 更新消息状态为已发送                                   │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
用户A、用户B收到消息
```

### 群聊消息流程

```
用户A发送群消息
      │
      ▼
   (同上步骤1-3)
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 群聊转发                                                  │
│    - 判断 receiveId[0] == 'G' (群聊)                       │
│    - 查询群成员列表 (GroupInfo.Members)                     │
│    - 遍历所有成员:                                          │
│      - 在线: Client.SendBack <- messageBack                │
│    - 更新Redis缓存                                          │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
所有在线群成员收到消息
```

## 七、配置切换

只需修改配置文件，无需改代码：

```toml
# config_local.toml

[kafkaConfig]
messageMode = "channel"   # 小规模，使用内存Channel
# messageMode = "kafka"   # 大规模，使用Kafka消息队列
```

**切换后的行为变化：**

| 配置值 | Client.Read() | Server启动 |
|--------|--------------|-----------|
| `channel` | 写入 `ChatServer.Transmit` | `ChatServer.Start()` |
| `kafka` | 写入 `Kafka Topic` | `KafkaChatServer.Start()` |

## 九、总结

| 文档 | 核心内容 | 创建的文件 |
|------|---------|-----------|
| **文档20** | 消息中转基础设施 | kafka_service.go, server.go, kafka_server.go, 枚举, DTO |
| **文档21** | 实时通信核心逻辑 | client.go (Read/Write/初始化/登出) |

**关系：**
- 文档20提供了"基础设施"（消息如何中转）
- 文档21提供了"业务逻辑"（WebSocket如何读写、消息如何处理）

**选择模式：**
- 小规模项目 → `messageMode = "channel"`
- 大规模项目 → `messageMode = "kafka"`
