# 教学文档 05: 日志系统（Zap）

### 日志系统是基础设施层

日志系统贯穿整个项目，在所有层都可以使用。按开发顺序：

```
Model → DAO → Service → Controller → 路由注册

日志系统可在任何阶段引入，主要用于记录：
- DAO层: 数据库连接/查询错误
- Service层: 业务逻辑异常
- Controller层: 请求参数错误
```

## 一、为什么需要日志？

### 日志的作用

1. **调试** - 掾障时查看运行流程
2. **监控** - 了解系统运行状态
3. **审计** - 记录用户操作历史

### 为什么用Zap？

| 特性 | Zap | log标准库 |
|------|-----|----------|
| 性能 | 极快 | 一般 |
| 结构化 | JSON格式 | 纯文本 |
| 级别 | INFO/WARN/ERROR/FATAL/DEBUG | 无级别 |

## 二、Zap日志库文件

**文件位置:** `pkg/zlog/logger.go`

根据文档 `4.后端开发.md` 行180-216：

### 完整代码（带详细注释）

```go
package zlog

// ============================================================
// 导入依赖包
// ============================================================
import (
    "os"
    "path"
    "runtime"
    
    // ★Zap日志库
    "go.uber.org/zap"
    "go.uber.org/zap/zapcore"
    
    // ★项目内部依赖
    "kama-chat-server/internal/config"
)

// ============================================================
// 全局Logger实例
// ============================================================
var logger *zap.Logger
var logPath string

// ============================================================
// init() 函数 - 自动初始化日志
// ============================================================
func init() {
    // 1. 获取配置中的日志路径
    conf := config.GetConfig()
    logPath = conf.LogConfig.LogPath
    
    // 2. 配置编码器（日志格式）
    encoderConfig := zap.NewProductionEncoderConfig()
    
    // ★时间格式: ISO8601（如 2024-01-16T15:30:00+08:00）
    encoderConfig.EncodeTime = zapcore.ISO8601TimeEncoder
    
    // ★创建JSON编码器
    // JSON格式方便日志分析工具解析
    encoder := zapcore.NewJSONEncoder(encoderConfig)
    
    // 3. 创建多输出流
    // ★同时输出到控制台和文件
    
    // 控制台输出
    consoleWriteSyncer := zapcore.AddSync(os.Stdout)
    
    // 文件输出
    file, err := os.OpenFile(logPath, os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644)
    if err != nil {
        panic("日志文件创建失败: " + err.Error())
    }
    fileWriteSyncer := zapcore.AddSync(file)
    
    // 4. 创建Core
    // ★zapcore.NewTee: 同时使用多个输出流
    core := zapcore.NewTee(
        zapcore.NewCore(encoder, consoleWriteSyncer, zapcore.DebugLevel), // 控制台
        zapcore.NewCore(encoder, fileWriteSyncer, zapcore.DebugLevel),    // 文件
    )
    
    // 5. 创建Logger
    // ★zap.AddCaller: 添加调用信息（函数名、文件名、行号）
    logger = zap.New(core, zap.AddCaller())
}

// ============================================================
// getCallerInfoForLog - 获取调用信息
// ============================================================
// ★runtime.Caller(skip): 回溯调用栈
// skip=0: 当前函数
// skip=1: 调用当前函数的函数
// skip=2: 再上一级（实际的日志调用位置）
func getCallerInfoForLog() []zap.Field {
    pc, file, line, ok := runtime.Caller(2)  // ★回溯两层
    if !ok {
        return nil
    }
    
    // 获取函数名
    funcName := path.Base(runtime.FuncForPC(pc).Name())
    
    // 返回zap.Field数组
    return []zap.Field{
        zap.String("func", funcName),  // 函数名
        zap.String("file", file),      // 文件名
        zap.Int("line", line),         // 行号
    }
}

// ============================================================
// Info - 信息级别日志
// ============================================================
func Info(message string, fields ...zap.Field) {
    fields = append(fields, getCallerInfoForLog()...)
    logger.Info(message, fields...)
}

// ============================================================
// Error - 错误级别日志
// ============================================================
func Error(message string, fields ...zap.Field) {
    fields = append(fields, getCallerInfoForLog()...)
    logger.Error(message, fields...)
}

// ============================================================
// Warn - 警告级别日志
// ============================================================
func Warn(message string, fields ...zap.Field) {
    fields = append(fields, getCallerInfoForLog()...)
    logger.Warn(message, fields...)
}

// ============================================================
// Fatal - 致命错误日志（会终止程序）
// ============================================================
func Fatal(message string, fields ...zap.Field) {
    fields = append(fields, getCallerInfoForLog()...)
    logger.Fatal(message, fields...)
}

// ============================================================
// Debug - 调试级别日志
// ============================================================
func Debug(message string, fields ...zap.Field) {
    fields = append(fields, getCallerInfoForLog()...)
    logger.Debug(message, fields...)
}
```

## 三、日志级别详解

### 级别说明

| 级别 | 用途 | 示例 |
|------|------|------|
| Debug | 调试信息 | 变量值、流程跟踪 |
| Info | 正常信息 | API请求、用户登录 |
| Warn | 警告信息 | 配置缺失、性能下降 |
| Error | 错误信息 | 数据库错误、API失败 |
| Fatal | 致命错误 | 启动失败（会终止程序） |

### 级别优先级

```
Debug < Info < Warn < Error < Fatal

设置级别为Info: Debug不会输出，其他都会输出
设置级别为Debug: 所有都会输出
```

## 四、日志输出格式

### JSON格式示例

```json
{
    "level": "info",
    "ts": "2024-01-16T15:30:00+08:00",
    "msg": "用户登录成功",
    "func": "Login",
    "file": "user_info_controller.go",
    "line": 25,
    "uuid": "U12345678901"
}
```

### 字段说明

| 字段 | 含义 |
|------|------|
| level | 日志级别 |
| ts | 时间戳 |
| msg | 日志消息 |
| func | 函数名 |
| file | 文件名 |
| line | 行号 |

## 五、使用示例

### 基本使用

```go
package main

import "kama-chat-server/pkg/zlog"

func Login() {
    zlog.Info("用户登录成功")
    zlog.Error("密码验证失败")
}
```

### 带字段使用

```go
// ★zap.String/Stringer添加额外信息
zlog.Info("用户登录成功", 
    zap.String("uuid", user.Uuid),
    zap.String("telephone", user.Telephone),
)

zlog.Error("数据库查询失败",
    zap.String("error", err.Error()),
    zap.String("query", "SELECT * FROM user_info"),
)
```

### 常用zap.Field

```go
zap.String("key", "value")     // 字符串
zap.Int("count", 100)          // 整数
zap.Bool("success", true)      // 布尔
zap.Duration("time", 5*time.Second) // 时间间隔
zap.Any("data", struct{}{})    // 任意类型
```

## 六、runtime.Caller详解

### 回溯原理

```
调用栈:
0: getCallerInfoForLog()     ← runtime.Caller(0)获取这里
1: Info()                    ← runtime.Caller(1)获取这里
2: 实际调用位置 Login()       ← runtime.Caller(2)获取这里 ★
```

### 获取的信息

```go
pc, file, line, ok := runtime.Caller(2)

// pc: 程序计数器（用于获取函数名）
// file: 文件路径（如 user_info_controller.go）
// line: 行号（如 25）
// ok: 是否成功获取
```

## 八、文件权限说明

```go
os.OpenFile(logPath, os.O_CREATE|os.O_APPEND|os.O_WRONLY, 0644)
```

| 标志 | 作用 |
|------|------|
| `O_CREATE` | 文件不存在则创建 |
| `O_APPEND` | 追加写入（不覆盖） |
| `O_WRONLY` | 只写模式 |
| `0644` | 权限（所有者读写，其他人只读） |
