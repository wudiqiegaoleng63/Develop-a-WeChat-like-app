# 教学文档 17: 联系人申请流程

### 开发顺序（重要）

```
Model → DAO → Service → Controller → 路由注册

① Model层: 联系人申请模型定义（contact_apply.go）
② DAO层: 数据库连接已在前序文档完成
③ Service层: 申请流程业务逻辑实现（user_contact_service.go）
④ Controller层: HTTP请求处理（user_contact_controller.go）
⑤ 路由注册: 在https_server中注册路由
```

## 流程概述

联系人申请流程是即时通讯系统中的核心功能，用于处理用户之间的好友添加请求以及用户申请加入群聊的审核流程。本模块涉及以下核心接口：

| 接口 | 功能 | 核心要点 |
|------|------|----------|
| POST /contact/applyContact | 申请添加联系人 | 判断用户/群聊类型（ContactId[0]=='U'或'G'） |
| POST /contact/getNewContactList | 获取新联系人申请列表 | 查询状态为PENDING的申请记录 |
| POST /contact/passContactApply | 通过联系人申请 | 创建双向UserContact记录（用户类型） |
| POST /contact/refuseContactApply | 拒绝联系人申请 | 更新申请状态为REFUSE |
| POST /contact/blackApply | 拉黑申请人 | 更新申请状态为BLACK |
| POST /contact/getAddGroupList | 获取进群申请列表 | 群主审核进群申请 |

## 1. POST /contact/applyContact - 申请添加联系人

### 接口概述

申请添加联系人接口用于用户发起添加好友或申请加入群聊的请求。该接口需要根据ContactId的首字符判断目标类型：
- `ContactId[0] == 'U'` 表示申请添加用户好友
- `ContactId[0] == 'G'` 表示申请加入群聊

申请记录的Uuid格式为：`"A" + 11位随机字符`，例如 `"A12345678901"`。

### Request结构体定义

```go
// internal/dto/request/apply_contact_request.go
package request

type ApplyContactRequest struct {
    OwnerId   string `json:"owner_id"`   // 申请人的用户uuid
    ContactId string `json:"contact_id"` // 被申请的联系人uuid（用户或群聊）
    Message   string `json:"message"`    // 申请信息/申请理由
}
```

### Respond结构体定义

该接口成功时只返回message，无data字段。

```go
// 成功响应
{
    "code": 200,
    "message": "申请成功"
}

// 失败响应示例
{
    "code": 400,
    "message": "用户不存在"
}
{
    "code": 400,
    "message": "对方已将你拉黑"
}
```

### Service层完整代码

```go
// internal/service/gorm/user_contact_service.go
package gorm

import (
    "errors"
    "fmt"
    "time"
    "gorm.io/gorm"
    "gochat/internal/dao"
    "gochat/internal/dto/request"
    "gochat/internal/model"
    "gochat/pkg/constants"
    "gochat/pkg/enum/contact/contact_type_enum"
    "gochat/pkg/enum/contact_apply/contact_apply_status_enum"
    "gochat/pkg/enum/group_info/group_status_enum"
    "gochat/pkg/enum/user_info/user_status_enum"
    "gochat/pkg/util/random"
    "gochat/pkg/zlog"
)

// ApplyContact 申请添加联系人
func (u *userContactService) ApplyContact(req request.ApplyContactRequest) (string, int) {
    // 判断联系人类型：用户类型（ContactId首字符为'U'）
    if req.ContactId[0] == 'U' {
        // 1. 检查目标用户是否存在
        var user model.UserInfo
        if res := dao.GormDB.First(&user, "uuid = ?", req.ContactId); res.Error != nil {
            if errors.Is(res.Error, gorm.ErrRecordNotFound) {
                zlog.Error("用户不存在")
                return "用户不存在", -2
            } else {
                zlog.Error(res.Error.Error())
                return constants.SYSTEM_ERROR, -1
            }
        }

        // 2. 检查目标用户是否被禁用
        if user.Status == user_status_enum.DISABLE {
            zlog.Info("用户已被禁用")
            return "用户已被禁用", -2
        }

        // 3. 查询是否已存在申请记录
        var contactApply model.ContactApply
        if res := dao.GormDB.Where("user_id = ? AND contact_id = ?", req.OwnerId, req.ContactId).First(&contactApply); res.Error != nil {
            if errors.Is(res.Error, gorm.ErrRecordNotFound) {
                // 4. 不存在申请记录，创建新的申请记录
                value, err := random.GetNowAndLenRandomString(11)
                if err != nil {
                    zlog.Error(err.Error())
                    return constants.SYSTEM_ERROR, -1
                }
                contactApply = model.ContactApply{
                    Uuid:        fmt.Sprintf("A%s", value),
                    UserId:      req.OwnerId,
                    ContactId:   req.ContactId,
                    ContactType: contact_type_enum.USER,
                    Status:      contact_apply_status_enum.PENDING,
                    Message:     req.Message,
                    LastApplyAt: time.Now(),
                }
                if res := dao.GormDB.Create(&contactApply); res.Error != nil {
                    zlog.Error(res.Error.Error())
                    return constants.SYSTEM_ERROR, -1
                }
            } else {
                zlog.Error(res.Error.Error())
                return constants.SYSTEM_ERROR, -1
            }
        }

        // 5. 如果存在申请记录，检查是否被拉黑
        if contactApply.Status == contact_apply_status_enum.BLACK {
            return "对方已将你拉黑", -2
        }

        // 6. ★重置申请状态为PENDING，更新申请时间（支持多次申请）
        contactApply.Status = contact_apply_status_enum.PENDING
        contactApply.LastApplyAt = time.Now()
        if res := dao.GormDB.Save(&contactApply); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }
        return "申请成功", 0

    } else if req.ContactId[0] == 'G' {
        // 群聊类型处理流程
        // 1. 检查群聊是否存在
        var group model.GroupInfo
        if res := dao.GormDB.First(&group, "uuid = ?", req.ContactId); res.Error != nil {
            if errors.Is(res.Error, gorm.ErrRecordNotFound) {
                zlog.Error("群聊不存在")
                return "群聊不存在", -2
            } else {
                zlog.Error(res.Error.Error())
                return constants.SYSTEM_ERROR, -1
            }
        }

        // 2. 检查群聊是否被禁用
        if group.Status == group_status_enum.DISABLE {
            zlog.Info("群聊已被禁用")
            return "群聊已被禁用", -2
        }

        // 3. 查询是否已存在申请记录
        var contactApply model.ContactApply
        if res := dao.GormDB.Where("user_id = ? AND contact_id = ?", req.OwnerId, req.ContactId).First(&contactApply); res.Error != nil {
            if errors.Is(res.Error, gorm.ErrRecordNotFound) {
                // 4. 不存在申请记录，创建新的申请记录
                value, err := random.GetNowAndLenRandomString(11)
                if err != nil {
                    zlog.Error(err.Error())
                    return constants.SYSTEM_ERROR, -1
                }
                contactApply = model.ContactApply{
                    Uuid:        fmt.Sprintf("A%s", value),
                    UserId:      req.OwnerId,
                    ContactId:   req.ContactId,
                    ContactType: contact_type_enum.GROUP,
                    Status:      contact_apply_status_enum.PENDING,
                    Message:     req.Message,
                    LastApplyAt: time.Now(),
                }
                if res := dao.GormDB.Create(&contactApply); res.Error != nil {
                    zlog.Error(res.Error.Error())
                    return constants.SYSTEM_ERROR, -1
                }
            } else {
                zlog.Error(res.Error.Error())
                return constants.SYSTEM_ERROR, -1
            }
        }

        // 5. 更新申请时间
        contactApply.LastApplyAt = time.Now()
        if res := dao.GormDB.Save(&contactApply); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }
        return "申请成功", 0

    } else {
        return "用户/群聊不存在", -2
    }
}
```

### 数据库操作说明

| 操作 | SQL语句（GORM翻译后） | 说明 |
|------|----------------------|------|
| 查询用户 | `SELECT * FROM user_info WHERE uuid = 'Uxxx' AND deleted_at IS NULL` | 检查目标用户是否存在 |
| 查询申请记录 | `SELECT * FROM contact_apply WHERE user_id = 'U1' AND contact_id = 'U2' AND deleted_at IS NULL` | 查找历史申请 |
| 创建申请记录 | `INSERT INTO contact_apply (...)` | 创建新的申请记录 |
| 更新申请时间 | `UPDATE contact_apply SET last_apply_at = NOW() WHERE id = ?` | 支持多次申请 |

### 关键概念详解

**1. Uuid生成规则**

申请记录的Uuid采用 `"A" + 11位随机字符` 的格式：
- 'A' 前缀表示这是一个申请记录（Apply）
- 后11位使用当前时间戳+随机字符串确保唯一性
- 例如：`A20240315123456` 或 `Aabc123def45`

```go
// random.GetNowAndLenRandomString 函数示例
func GetNowAndLenRandomString(length int) string {
    // 结合时间戳和随机字符生成唯一字符串
    timestamp := time.Now().Format("20060102150405")
    randomPart := getRandomString(length - len(timestamp))
    return timestamp + randomPart
}
```

**2. 类型判断机制**

通过检查ContactId的首字符来判断目标类型：
```go
if ContactId[0] == 'U' {
    // 用户类型处理
} else if ContactId[0] == 'G' {
    // 群聊类型处理
}
```

这种设计的好处：
- 简单直观，无需额外字段标识
- 快速判断，效率高
- 与UserInfo（'U'前缀）和GroupInfo（'G'前缀）的Uuid格式一致

**3. 拉黑状态检查**

如果之前申请被拉黑（status=BLACK），则无法再次申请：
```go
if contactApply.Status == contact_apply_status_enum.BLACK {
    return "对方已将你拉黑", -2
}
```

```go
// internal/service/gorm/user_contact_service.go

// GetNewContactList 获取新的联系人申请列表
func (u *userContactService) GetNewContactList(ownerId string) (string, []respond.NewContactListRespond, int) {
    // 1. 查询所有状态为PENDING且contact_id为当前用户的申请记录
    var contactApplyList []model.ContactApply
    if res := dao.GormDB.Where("contact_id = ? AND status = ?", ownerId, contact_apply_status_enum.PENDING).Find(&contactApplyList); res.Error != nil {
        if errors.Is(res.Error, gorm.ErrRecordNotFound) {
            zlog.Info("没有在申请的联系人")
            return "没有在申请的联系人", nil, 0
        } else {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, nil, -1
        }
    }

    // 2. 构建响应数据
    var rsp []respond.NewContactListRespond
    for _, contactApply := range contactApplyList {
        // 格式化申请信息
        var message string
        if contactApply.Message == "" {
            message = "申请理由：无"
        } else {
            message = "申请理由：" + contactApply.Message
        }

        newContact := respond.NewContactListRespond{
            Message: message,
        }

        // 3. 查询申请人的用户信息
        var user model.UserInfo
        if res := dao.GormDB.First(&user, "uuid = ?", contactApply.UserId); res.Error != nil {
            return constants.SYSTEM_ERROR, nil, -1
        }

        newContact.ContactId = user.Uuid
        newContact.ContactName = user.Nickname
        newContact.ContactAvatar = user.Avatar
        rsp = append(rsp, newContact)
    }
    return "获取成功", rsp, 0
}
```

### 数据库操作说明

| 操作 | SQL语句 | 说明 |
|------|---------|------|
| 查询申请列表 | `SELECT * FROM contact_apply WHERE contact_id = 'Uowner' AND status = 0 AND deleted_at IS NULL` | 查找所有待处理的申请 |
| 查询申请人信息 | `SELECT * FROM user_info WHERE uuid = 'Uapplicant' AND deleted_at IS NULL` | 获取申请人昵称和头像 |

### 关键概念详解

**查询条件解析**

```go
dao.GormDB.Where("contact_id = ? AND status = ?", ownerId, contact_apply_status_enum.PENDING)
```

- `contact_id = ownerId`：表示查询的是发给当前用户（被申请方）的申请
- `status = PENDING`：只查询状态为"申请中"的记录

这与申请接口中的字段含义相反：
- 申请时：`UserId`是申请人，`ContactId`是被申请人
- 查询时：查询条件`contact_id = ownerId`表示查询发给自己的申请

## 3. POST /contact/passContactApply - 通过联系人申请

### 接口概述

通过联系人申请接口是整个流程的核心，用于被申请方同意好友申请或群聊申请。该接口的关键特点是：

- **用户类型**：需要创建**两条**UserContact记录（双向好友关系）
- **群聊类型**：创建一条UserContact记录，并更新GroupInfo.Members

### Request结构体定义

```go
// internal/dto/request/pass_contact_apply_request.go
package request

type PassContactApplyRequest struct {
    OwnerId   string `json:"owner_id"`   // 被申请方uuid（用户或群聊）
    ContactId string `json:"contact_id"` // 申请人的用户uuid
}
```

**注意**：OwnerId的含义根据类型不同：
- 用户类型：OwnerId是登录用户的uuid
- 群聊类型：OwnerId是群聊的uuid（不是群主uuid）

### Respond结构体定义

成功时只返回message，无data字段。

```go
// 成功响应
{
    "code": 200,
    "message": "已添加该联系人"  // 用户类型
}

// 或
{
    "code": 200,
    "message": "已通过加群申请"  // 群聊类型
}
```

```go
// internal/service/gorm/user_contact_service.go

// PassContactApply 通过联系人申请
// ownerId 如果是用户的话就是登录用户，如果是群聊的话就是群聊id
func (u *userContactService) PassContactApply(ownerId string, contactId string) (string, int) {
    // 1. 查询申请记录
    var contactApply model.ContactApply
    if res := dao.GormDB.Where("contact_id = ? AND user_id = ?", ownerId, contactId).First(&contactApply); res.Error != nil {
        zlog.Error(res.Error.Error())
        return constants.SYSTEM_ERROR, -1
    }

    // 2. 根据OwnerId首字符判断类型
    if ownerId[0] == 'U' {
        // ===== 用户类型处理 =====
        
        // 2.1 检查申请人是否被禁用
        var user model.UserInfo
        if res := dao.GormDB.Where("uuid = ?", contactId).Find(&user); res.Error != nil {
            zlog.Error(res.Error.Error())
        }
        if user.Status == user_status_enum.DISABLE {
            zlog.Error("用户已被禁用")
            return "用户已被禁用", -2
        }

        // 2.2 更新申请状态为通过
        contactApply.Status = contact_apply_status_enum.AGREE
        if res := dao.GormDB.Save(&contactApply); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // 2.3 创建第一条UserContact（OwnerId -> ContactId）
        newContact := model.UserContact{
            UserId:      ownerId,
            ContactId:   contactId,
            ContactType: contact_type_enum.USER,     // 用户类型
            Status:      contact_status_enum.NORMAL, // 正常状态
            CreatedAt:   time.Now(),
            UpdateAt:    time.Now(),
        }
        if res := dao.GormDB.Create(&newContact); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // 2.4 创建第二条UserContact（ContactId -> OwnerId）
        // ★ 关键：用户好友关系需要双向记录
        anotherContact := model.UserContact{
            UserId:      contactId,
            ContactId:   ownerId,
            ContactType: contact_type_enum.USER,     // 用户类型
            Status:      contact_status_enum.NORMAL, // 正常状态
            CreatedAt:   newContact.CreatedAt,
            UpdateAt:    newContact.UpdateAt,
        }
        if res := dao.GormDB.Create(&anotherContact); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // ★ 删除Redis缓存
        if err := myredis.DelKeysWithPattern("contact_user_list_" + ownerId); err != nil {
            zlog.Error(err.Error())
        }
        return "已添加该联系人", 0

    } else {
        // ===== 群聊类型处理 =====
        
        // 3.1 检查群聊是否被禁用
        var group model.GroupInfo
        if res := dao.GormDB.Where("uuid = ?", ownerId).Find(&group); res.Error != nil {
            zlog.Error(res.Error.Error())
        }
        if group.Status == group_status_enum.DISABLE {
            zlog.Error("群聊已被禁用")
            return "群聊已被禁用", -2
        }

        // 3.2 更新申请状态为通过
        contactApply.Status = contact_apply_status_enum.AGREE
        if res := dao.GormDB.Save(&contactApply); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // 3.3 创建一条UserContact
        // ★ 关键：群聊只需要一条记录，因为一个UserContact足以表达双方状态
        newContact := model.UserContact{
            UserId:      contactId,   // 申请人的uuid
            ContactId:   ownerId,     // 群聊uuid
            ContactType: contact_type_enum.GROUP,    // 群聊类型
            Status:      contact_status_enum.NORMAL, // 正常状态
            CreatedAt:   time.Now(),
            UpdateAt:    time.Now(),
        }
        if res := dao.GormDB.Create(&newContact); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // 3.4 更新群聊成员列表
        var members []string
        if err := json.Unmarshal(group.Members, &members); err != nil {
            zlog.Error(err.Error())
            return constants.SYSTEM_ERROR, -1
        }
        members = append(members, contactId)  // 添加新成员
        
        // 3.5 更新群人数和成员列表
        group.MemberCnt = len(members)
        group.Members, _ = json.Marshal(members)
        if res := dao.GormDB.Save(&group); res.Error != nil {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, -1
        }

        // ★ 删除Redis缓存
        if err := myredis.DelKeysWithPattern("my_joined_group_list_" + ownerId); err != nil {
            zlog.Error(err.Error())
        }
        return "已通过加群申请", 0
    }
}
```

### 数据库操作说明

| 操作 | 类型 | SQL语句 | 说明 |
|------|------|---------|------|
| 查询申请记录 | 共用 | `SELECT * FROM contact_apply WHERE contact_id = ? AND user_id = ?` | 找到对应的申请 |
| 创建UserContact(1) | 用户 | `INSERT INTO user_contact (user_id='U1', contact_id='U2')` | 被申请方的好友列表 |
| 创建UserContact(2) | 用户 | `INSERT INTO user_contact (user_id='U2', contact_id='U1')` | 申请方的好友列表 |
| 创建UserContact | 群聊 | `INSERT INTO user_contact (user_id='Uuser', contact_id='Ggroup')` | 用户加入群聊 |
| 更新群成员 | 群聊 | `UPDATE group_info SET members=?, member_cnt=? WHERE uuid=?` | 添加群成员 |

### 关键概念详解

**1. 双向UserContact的重要性**

用户好友关系需要两条记录的原因：

```
用户A的好友列表: UserContact(userId='A', contactId='B')
用户B的好友列表: UserContact(userId='B', contactId='A')
```

好处：
- 每个用户可以独立管理自己的好友列表
- 支持单向操作（如拉黑、删除）不影响对方
- 查询好友列表时只需查询`user_id = 自己`的记录

**2. 群聊成员的JSON序列化**

GroupInfo.Members字段使用`json.RawMessage`存储：

```go
// 序列化（存储）
members := []string{"U001", "U002", "U003"}
group.Members, _ = json.Marshal(members)

// 反序列化（读取）
var members []string
json.Unmarshal(group.Members, &members)
```

这是因为GORM不支持直接存储`[]string`类型，需要序列化为`[]byte`。

**3. 群聊类型只需要一条UserContact**

```go
// 群聊联系人记录
UserContact{
    UserId:      "Uuser",   // 用户uuid
    ContactId:   "Ggroup",  // 群聊uuid
    ContactType: GROUP,
}
```

原因：
- 群聊不会主动添加用户，只有用户申请加入群聊
- 不需要群聊方管理"好友列表"
- 一条记录足以表达"用户在群聊中"的关系

```go
func (u *userContactService) GetAddGroupList(groupId string) (string, []respond.AddGroupListRespond, int) {

    var contactApplyList []model.ContactApply
    if res := dao.GormDB.Where("contact_id = ? AND status = ?", groupId, contact_apply_status_enum.PENDING).Find(&contactApplyList); res.Error != nil {
        if errors.Is(res.Error, gorm.ErrRecordNotFound) {

            return "没有在申请的联系人", nil, 0
        } else {
            zlog.Error(res.Error.Error())
            return constants.SYSTEM_ERROR, nil, -1
        }

    var rsp []respond.AddGroupListRespond
    for _, contactApply := range contactApplyList {
        // 格式化申请信息
        var message string
        if contactApply.Message == "" {
            message = "申请理由：无"
        } else {
            message = "申请理由：" + contactApply.Message
        }

            ContactId: contactApply.Uuid,  // 申请记录的uuid
            Message:   message,
        }

        var user model.UserInfo
        if res := dao.GormDB.First(&user, "uuid = ?", contactApply.UserId); res.Error != nil {
            return constants.SYSTEM_ERROR, nil, -1
        }

        // 4. 更新申请人信息
        newContact.ContactId = user.Uuid
        newContact.ContactName = user.Nickname
        newContact.ContactAvatar = user.Avatar
        rsp = append(rsp, newContact)
    }
    return "获取成功", rsp, 0
}
```

## 完整流程总结

### 申请状态流转

```
初始状态: 无申请记录
    │
    ▼ applyContact
状态: PENDING (申请中)
    │
    ├──────────────────┬──────────────────┬─────────────────
    │                  │                  │
    ▼                  ▼                  ▼
passContactApply    refuseContactApply  blackApply
    │                  │                  │
    ▼                  ▼                  ▼
状态: AGREE (通过)   状态: REFUSE (拒绝)  状态: BLACK (拉黑)
    │                  │                  │
    ▼                  ▼                  ▼
创建UserContact      可再次申请         无法再次申请
```

### UserContact创建规则

| 场景 | UserContact数量 | 记录详情 |
|------|----------------|----------|
| 用户通过好友申请 | 2条 | UserId=A,ContactId=B 和 UserId=B,ContactId=A |
| 用户通过群聊申请 | 1条 | UserId=申请人,ContactId=群聊uuid |
| 拒绝申请 | 0条 | 不创建任何记录 |
| 拉黑申请 | 0条 | 不创建任何记录 |

### 类型判断汇总表

| 接口 | 判断方式 | 处理逻辑 |
|------|----------|----------|
| applyContact | `ContactId[0]` | 'U'用户申请，'G'群聊申请 |
| passContactApply | `OwnerId[0]` | 'U'创建双向记录，'G'更新群成员 |
| refuseContactApply | `OwnerId[0]` | 仅返回不同提示信息 |
| blackApply | 无需判断 | 仅更新申请状态 |

## 常见问题与解决方案

### Q1: 为什么用户好友关系需要两条UserContact记录？

**答案**：因为好友关系需要双向管理：
- 用户A删除好友，只影响A的列表，不影响B
- 用户A拉黑好友，状态记录在A的记录中（BLACK），B的记录中（BE_BLACK）
- 查询好友列表时只需查询`user_id = 自己`，效率更高

### Q2: 申请被拒绝后可以再次申请吗？

**答案**：可以。REFUSE状态只是记录拒绝结果，不影响后续申请。再次申请时会更新`LastApplyAt`时间。但BLACK状态会阻止后续申请。

### Q3: 群聊申请和好友申请有什么区别？

**答案**：
| 对比项 | 好友申请 | 群聊申请 |
|--------|----------|----------|
| ContactType | USER(0) | GROUP(1) |
| 审核人 | 被申请用户 | 群主 |
| 通过后操作 | 创建2条UserContact | 创建1条UserContact + 更新群成员 |

### Q4: 申请Uuid为什么要用"A"前缀？

**答案**：便于快速识别数据类型：
- 'U'前缀：用户UserInfo
- 'G'前缀：群聊GroupInfo
- 'A'前缀：申请ContactApply
- 'S'前缀：会话Session
- 'M'前缀：消息Message

这种设计使得UUID自带类型信息，便于调试和日志分析。
