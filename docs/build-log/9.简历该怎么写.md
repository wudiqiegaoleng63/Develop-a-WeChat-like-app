### 可能涉及的项目难点
+ **多技术深度融合挑战大**：项目运用 gin、gorm、webrtc 等多种技术栈，需打通各技术间的通信与数据流转，实现无缝协同工作，确保系统稳定运行。
+ **跨域与安全保障复杂**：要实现 cors 跨域资源共享和 recovery 恢复 err 中间件解决跨域问题，同时运用 sms 验证和 ssl 加密保障系统安全，防止信息泄露与恶意攻击。
+ **数据库设计与优化不易**：基于 gorm 搭建数据库，使用 redis 作为查询中间层，用户群组表采用软删除，需维护数据一致性和高性能，设计与优化难度大。
+ **音视频功能实现困难多**：采用 webrtc 实现音视频通话，要处理媒体编解码、信令交互等问题，实现 1 对 1 通话逻辑，确保通话质量稳定。
+ **一人承担前后端开发压力重**：独自负责前后端开发，需平衡两端开发进度，解决前后端交互问题，保障系统整体的用户体验和功能完整性。

### 可能涉及的个人收获
+ 对后端的 gin 路由框架运用自如，熟练搭建路由系统并配置中间件，解决跨域与错误恢复问题。
+ 深入掌握 gorm 进行数据库操作，能设计合理表结构，结合 redis 优化查询性能，熟悉软删除机制。
+ 熟悉 websocket 原理，能基于其构建高效的聊天服务器，处理单聊、群聊及登录登出消息。
+ 前端方面，精通 vue 及其生态系统，借助 vue-router 管理路由，用 vuex 实现状态管理与用户信息存储。
