## 项目优势？
### 市场go项目相对于java/cpp没有这么多
在当下的软件开发市场中，Java 和 C++ 项目占据了相当大的比重，它们凭借着成熟的技术生态和广泛的应用场景，长期以来都是开发者们的主流选择。相比之下，Go 语言的项目数量看似没有那么多。然而，近年来，Go 语言凭借其简洁高效的语法、出色的并发处理能力以及快速的编译速度，在各大互联网大厂中逐渐崭露头角，成为一门至关重要的开发语言。越来越多的大厂开始在核心业务中采用 Go 语言，例如在云计算、分布式系统、微服务架构等领域，Go 语言都有着出色的表现。因此，为了紧跟技术发展的趋势，提升自身在软件开发领域的竞争力，我们有必要通过参与 Go 项目来深入掌握和应用这门语言。

我们所参与的这个 Go 项目，**涉及的技术面极为广泛**，尤其是从后端开发的角度来看，构建了一个庞大且复杂的技术体系。

首先是后台开发者管理模块。这一模块对于整个项目的稳定运行和高效开发起着至关重要的作用。它涵盖了用户权限管理、开发者账号安全、操作日志记录等多个方面。通过合理的权限分配，不同级别的开发者可以对系统进行不同程度的操作，确保敏感信息和关键功能的安全性。同时，操作日志的记录可以方便后续的问题排查和审计工作，保证系统的可追溯性。



### 涉及技术面广
此项目构建了一个全面且复杂的即时通讯系统，前后端融合了广泛的技术体系，深入研究各个技术模块，能让人对即时通讯领域形成系统而完整的认知。下面将详细阐述各个技术板块及其具体实现功能：

#### 1. 后台开发者管理
这一模块赋予后台开发者强大的管理权限，主要用于对用户群聊和用户角色进行管控。开发者可以对用户群聊执行禁用、启用和删除操作，以确保群聊的正常秩序和合规性。同时，开发者还能根据实际需求，将普通用户设置为管理员，协助进行系统管理。

#### 2. 类似聊天软件的联系人体系
该体系模拟了常见聊天软件的联系人管理功能，为用户提供了丰富的社交互动选项。用户可以自由添加或删除联系人，还能对联系人进行拉黑操作。当用户想要添加新联系人时，可以发起申请，对方则有权选择同意或拒绝该申请。

#### 3. 单聊与群聊
单聊和群聊是即时通讯系统的核心功能之一，其实现依赖于后端的聊天服务器。无论是单聊消息还是群聊消息，都会被发送到后端服务器，由服务器负责将消息准确无误地转发给相应的接收方。

#### 4. 多种消息类型的上传和下载（文本、文件、视频）
系统支持多种类型的消息交互，包括文本、文件和视频。用户可以方便地上传和下载这些不同类型的消息，满足多样化的沟通需求。

#### 5. Kafka
Kafka 在项目中扮演着消息传输的重要角色。它作为一个高效的消息队列系统，负责将客户端（client）发送的消息转发到服务器（server），确保消息的可靠传输和处理。

#### 6. 语音视频通话
语音视频通话功能为用户提供了更加直观和实时的沟通方式。用户可以发起通话邀请，对方可以选择接受或拒绝邀请。在通话过程中，任何一方都可以随时挂断通话，结束交流。

#### 7. WebSocket
WebSocket 为前端和客户端之间建立了实时、双向的通信连接。它负责接收前端发送给客户端的消息，并将客户端的消息传递给服务器，实现了消息的高效传输和即时响应。

通过参与这个项目，对上述每个技术点进行深入钻研，开发者可以全面了解即时通讯系统的架构、原理和实现细节，从而在该领域积累丰富的经验和技能。



### 这是一个前后端分离的项目
前端使用vue，后端使用go。

对于开发者而言，采用前端 Vue、后端 Go 进行前后端分离开发的练习项目，有一些显著的意义：

**深入理解前端框架**：Vue 具有简洁的语法和高效的响应式系统。在练习项目中使用 Vue，开发者能深入掌握组件化开发思想，学会如何构建可复用的组件，以及运用 Vue Router 实现单页面应用的路由管理、Vuex 进行状态管理。这有助于开发者熟练运用现代前端开发技术，提升界面交互设计和用户体验优化能力。

**熟悉后端语言与框架**：Go 语言以其出色的并发性能和简洁的语法受到青睐。通过后端使用 Go 开发，开发者可以学习 Go 的标准库和常用框架，如 Gin、Gorm 等，掌握后端服务的搭建、路由配置、中间件使用等技能，了解如何处理 HTTP 请求、操作数据库以及实现业务逻辑。

**掌握前后端交互技术**：前后端分离要求开发者掌握前后端交互的技术，如 RESTful API 设计与使用。开发者需要学习如何设计合理的 API 接口，处理请求参数和响应数据，以及使用 JSON 进行数据传输。这有助于提升开发者对数据交互的理解和实践能力。

**代码维护与扩展**：前后端分离使得代码结构更加清晰，前端和后端的代码分别维护，降低了代码的耦合度。当项目需要进行扩展或修改时，开发者可以更容易地定位和修改问题代码，提高代码的可维护性。在练习项目中，开发者可以学习如何设计合理的代码结构，提高代码的质量和可维护性。
