# 教学文档 02: 配置管理（TOML）

## 一、TOML配置文件介绍

### 什么是TOML？

TOML (Tom's Obvious Minimal Language) 是一种配置文件格式，比JSON更易读，支持注释。

### 为什么用TOML而不是JSON？

| 特性 | TOML | JSON |
|------|------|------|
| 注释 | ✓ 支持 `# 注释` | ✗ 不支持 |
| 可读性 | ✓ 更易读 | 一般 |
| 类型明确 | ✓ 类型明确 | 需推断 |

## 二、TOML文件结构

根据文档 `4.后端开发.md` 行180-216，当前阶段只需要这3个配置：

**文件位置:** `configs/config_local.toml`

```toml
# ============================================================
# kama-chat-server 配置文件
# ============================================================

# ----------------------
# [mainConfig] 主配置 - 服务器基本参数
# ----------------------
[mainConfig]
appName = "kama-chat"        # 应用名称
host = "127.0.0.1"           # 监听地址
port = 8000                  # HTTP端口

# ----------------------
# [mysqlConfig] MySQL数据库配置
# ----------------------
[mysqlConfig]
host = "127.0.0.1"           # MySQL服务器地址
port = 3306                  # MySQL端口（默认3306）
user = "root"                # MySQL用户名
password = "你的密码"         # MySQL密码
databaseName = "kama_chat"   # 数据库名

# ----------------------
# [logConfig] 日志配置
# ----------------------
[logConfig]
logPath = "./logs/app.log"   # 日志文件路径
```

### TOML语法要点

```toml
# 1. 区块定义
[sectionName]
key = "value"

# 2. 字符串用双引号
appName = "kama-chat"

# 3. 数字直接写
port = 8000

# 4. 注释用 # 开头
# 这是注释
```

## 三、Go配置解析代码

**文件位置:** `internal/config/config.go`

### 完整代码（带详细注释）

```go
package config

// ============================================================
// 导入依赖包
// ============================================================
import (
    "log"
    
    // ★TOML解析库
    // 需要先安装: go get github.com/BurntSushi/toml
    "github.com/BurntSushi/toml"
)

// ============================================================
// 配置结构体定义
// ============================================================
// `toml:"xxx"` 标签：告诉解析器对应TOML中的哪个字段

// MainConfig - 主配置（服务器基本信息）
type MainConfig struct {
	AppName string `toml:"appName"` // 应用名称，用于日志标识
	Host    string `toml:"host"`    // 服务器监听地址
	Port    int    `toml:"port"`    // HTTP端口
}

// MysqlConfig - MySQL数据库配置
type MysqlConfig struct {
	Host         string `toml:"host"`         // MySQL服务器地址
	Port         int    `toml:"port"`         // MySQL端口，默认3306
	User         string `toml:"user"`         // MySQL用户名
	Password     string `toml:"password"`     // MySQL密码
	DatabaseName string `toml:"databaseName"` // 数据库名
}

// LogConfig - 日志配置
type LogConfig struct {
	LogPath string `toml:"logPath"` // 日志文件存储路径
}

// RedisConfig - Redis配置
type RedisConfig struct {
	Host     string `toml:"host"`     // Redis服务器地址
	Port     int    `toml:"port"`     // Redis端口，默认6379
	Password string `toml:"password"` // Redis密码
	Db       int    `toml:"db"`       // Redis数据库编号
}

// StaticSrcConfig - 静态资源配置
type StaticSrcConfig struct {
	StaticAvatarPath string `toml:"staticAvatarPath"` // 头像存储路径
	StaticFilePath   string `toml:"staticFilePath"`   // 文件存储路径
}

// EmailConfig - 邮箱验证码配置（QQ SMTP）
type EmailConfig struct {
	SmtpHost     string `toml:"smtpHost"`     // SMTP服务器（smtp.qq.com）
	SmtpPort     int    `toml:"smtpPort"`     // 端口（SSL用465）
	SmtpUsername string `toml:"smtpUsername"` // 发件邮箱
	SmtpPassword string `toml:"smtpPassword"` // 16位授权码（不是QQ密码）
	FromName     string `toml:"fromName"`     // 发件人名称
}

// ============================================================
// Config - 总配置结构体
// ============================================================
// ★匿名嵌入语法：Config"继承"了所有子配置的字段
// 访问方式：config.MainConfig.Host 或 config.MysqlConfig.Host
type Config struct {
	MainConfig      MainConfig      `toml:"mainConfig"`
	MysqlConfig     MysqlConfig     `toml:"mysqlConfig"`
	LogConfig       LogConfig       `toml:"logConfig"`
	StaticSrcConfig StaticSrcConfig `toml:"staticSrcConfig"`
	RedisConfig     RedisConfig     `toml:"redisConfig"`
	EmailConfig     EmailConfig     `toml:"emailConfig"`
}

// ============================================================
// 全局配置变量 - 单例模式
// ============================================================
// ★单例模式：全局只有一个config实例，避免重复加载
var config *Config

// ============================================================
// LoadConfig - 从TOML文件加载配置
// ============================================================
func LoadConfig() error {
    // ★toml.DecodeFile 函数：
    // 参数1: TOML文件路径
    // 参数2: 目标结构体指针（必须是指针！）
    // 返回: 解析结果和错误
    _, err := toml.DecodeFile("configs/config_local.toml", config)
    if err != nil {
        // 配置加载失败是致命错误，用log.Fatal终止程序
        log.Fatal("配置加载失败: ", err.Error())
        return err
    }
    return nil
}

// ============================================================
// GetConfig - 获取配置实例（懒加载）
// ============================================================
// ★懒加载模式：第一次调用时才加载，之后返回同一个实例
// 项目中任何地方需要配置，都调用这个函数
func GetConfig() *Config {
    if config == nil {
        // 第一次调用：创建结构体并加载配置
        config = new(Config)
        _ = LoadConfig()
    }
    // 后续调用：直接返回已加载的实例
    return config
}
```

## 四、关键概念详解

### 1. `toml:"xxx"` 标签的作用

```go
AppName string `toml:"appName"`
```

| 不写标签 | 写标签 | 结果 |
|---------|-------|------|
| GORM用驼峰名匹配 | 强制匹配指定名称 | TOML中的 `appName` 对应 Go中的 `AppName` |

### 2. 指针类型 `var config *Config`

```go
var config *Config  // config存储的是Config的地址，不是Config本身
```

**为什么用指针？**
- 可以判断是否已初始化：`if config == nil`
- 普通结构体无法用 `== nil` 判断

**指针详解：**

```go
// 声明指针（此时是nil）
var config *Config
fmt.Println(config)  // 输出: <nil>

// 用new创建实例
config = new(Config)
fmt.Println(config)  // 输出: &{主配置值 MySQL配置值}

// Go自动解引用，可以直接访问字段
config.MainConfig.Host = "127.0.0.1"
```

### 3. `new(Config)` 的作用

```go
config = new(Config)
```

做了两件事：
1. 在内存中创建一个 `Config` 结构体
2. 返回这个结构体的**地址**

等价于：
```go
var cfg Config        // 创建结构体
config = &cfg         // 取地址赋给指针
```

### 4. 单例模式

```go
var config *Config  // 全局变量

func GetConfig() *Config {
    if config == nil {      // 只在第一次调用时加载
        config = new(Config)
        LoadConfig()
    }
    return config           // 之后每次都返回同一个实例
}
```

**为什么用单例？**
- 配置只需要加载一次
- 多次加载浪费资源
- 确保配置数据一致性

### 5. 懒加载

```go
if config == nil {
    config = new(Config)
    LoadConfig()
}
```

**懒加载的好处：**
- 不用每次启动都加载
- 只有真正需要时才加载
- 减少启动时间

## 五、匿名嵌入（Embedding）

```go
type Config struct {
    MainConfig  MainConfig  `toml:"mainConfig"`
    MysqlConfig MysqlConfig `toml:"mysqlConfig"`
}
```

这不是继承，是组合。访问方式：

```go
cfg := config.GetConfig()

// 方式1: 通过子结构体访问
cfg.MainConfig.Host

// 方式2: 直接访问（Go自动解引用）
cfg.Host  // 等同于 cfg.MainConfig.Host
```

## 六、使用示例

```go
package main

import (
    "fmt"
    "kama-chat-server/internal/config"
)

func main() {
    // 获取配置
    cfg := config.GetConfig()
    
    // 使用配置
    fmt.Println("应用名称:", cfg.MainConfig.AppName)
    fmt.Println("服务器地址:", cfg.MainConfig.Host)
    fmt.Println("MySQL地址:", cfg.MysqlConfig.Host)
    fmt.Println("数据库名:", cfg.MysqlConfig.DatabaseName)
}
```
