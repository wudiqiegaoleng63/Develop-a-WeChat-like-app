## 一、聊天室功能概述

### 什么是聊天室？

聊天室是一个**临时的多人通信场景**，数据存储在内存中，不持久化。

### 聊天室 vs 群聊

| 特性 | 聊天室 | 群聊 |
|------|--------|------|
| 持久化 | 内存存储（不持久化） | 数据库持久化 |
| 成员管理 | 临时性，随时加入/退出 | 固定成员列表 |
| 消息存储 | 不存储历史消息 | 消息表持久存储 |
| 适用场景 | 即时多人讨论、游戏房间 | 长期多人沟通 |

## 五、Controller层代码

**文件位置**: `api/v1/chatroom_controller.go`

```go
package v1

import (
	"github.com/gin-gonic/gin"
	"kama_chat_server/internal/dto/request"
	"kama_chat_server/internal/service/gorm"
	"kama_chat_server/pkg/constants"
	"kama_chat_server/pkg/zlog"
	"net/http"
)

// GetCurContactListInChatRoom 获取当前聊天室联系人列表
func GetCurContactListInChatRoom(c *gin.Context) {
	var req request.GetCurContactListInChatRoomRequest
	if err := c.BindJSON(&req); err != nil {
		zlog.Error(err.Error())
		c.JSON(http.StatusOK, gin.H{
			"code":    500,
			"message": constants.SYSTEM_ERROR,
		})
		return
	}
	message, rspList, ret := gorm.ChatRoomService.GetCurContactListInChatRoom(req.OwnerId, req.ContactId)
	JsonBack(c, message, ret, rspList)
}
```

## 六、Service层代码

**文件位置**: `internal/service/gorm/chatroom_service.go`

```go
package gorm

import "kama_chat_server/internal/dto/respond"

type chatRoomService struct {
}

var ChatRoomService = new(chatRoomService)

type chatRoomKey struct {
	ownerId   string
	contactId string
}

// map 类型是 {string, string}: []string该怎么写
var chatRooms = make(map[chatRoomKey][]string)

// GetCurContactListInChatRoom 获取当前聊天室联系人列表
func (c *chatRoomService) GetCurContactListInChatRoom(ownerId string, contactId string) (string, []respond.GetCurContactListInChatRoomRespond, int) {
	var rspList []respond.GetCurContactListInChatRoomRespond
	for _, contactId := range chatRooms[chatRoomKey{ownerId, contactId}] {
		rspList = append(rspList, respond.GetCurContactListInChatRoomRespond{
			ContactId: contactId,
		})
	}
	return "获取聊天室联系人列表成功", rspList, 0
}
```

## 七、数据存储结构

### 复合键Map

```go
type chatRoomKey struct {
    ownerId   string  // 创建者Uuid
    contactId string  // 联系人Uuid
}

var chatRooms = make(map[chatRoomKey][]string)
```

### 数据示例

```go
chatRooms = map[chatRoomKey][]string{
```

## 九、关键技术点

### 9.1 为什么用struct作为Map Key

```go
// ✅ struct作为map key（推荐）
type chatRoomKey struct {
    ownerId   string
    contactId string
}
var chatRooms = make(map[chatRoomKey][]string)

// ❌ 字符串拼接作为key（不推荐）
var chatRooms = make(map[string][]string)
// key = ownerId + "_" + contactId  // 容易出错
```

**原因**：Go的map key必须是可比较类型，struct只要所有字段可比较就能作为key。

### 9.2 为什么用内存存储

| 存储方式 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| 内存 | 快，无IO | 重启丢失 | 临时数据 |
| 数据库 | 慢，需要查询 | 持久化 | 重要数据 |
| Redis | 快，可持久化 | 需要额外服务 | 缓存数据 |

聊天室是临时数据，用内存最合适。
