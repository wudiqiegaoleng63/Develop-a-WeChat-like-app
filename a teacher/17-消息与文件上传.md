## 一、POST /message/getMessageList - 获取私聊消息列表

### 1.1 接口概述

获取两个用户之间的私聊聊天记录。该接口用于打开会话时加载历史消息，支持双向查询（即查询 A->B 和 B->A 的所有消息）。

**核心特点**：
- 按照创建时间 `created_at ASC` 升序排序，确保聊天记录按时间顺序展示
- 查询条件覆盖双向通信（send_id/receive_id 双向匹配）

### 1.2 Request结构体定义

```go
// internal/dto/request/message_request.go

// GetMessageListRequest 获取聊天记录请求
type GetMessageListRequest struct {
    UserOneId string `json:"user_one_id"`  // 用户一的uuid（通常是当前登录用户）
    UserTwoId string `json:"user_two_id"`  // 用户二的uuid（聊天对象）
}
```

### 1.3 Respond结构体定义

```go

    SendAvatar string `json:"send_avatar"`  // 发送者头像
    ReceiveId  string `json:"receive_id"`   // 接收者uuid

    Url        string `json:"url"`          // 文件url（文件消息）

    FileType   string `json:"file_type"`    // 文件类型
```

### 1.5 Service层完整代码

```go
// internal/service/gorm/message_service.go
package gorm

import (
    "kama-chat-server/internal/dto/respond"
    "kama-chat-server/pkg/constants"
    "kama-chat-server/internal/dao"
    "kama-chat-server/internal/model"
    "kama-chat-server/pkg/zlog"

    "github.com/gin-gonic/gin"
)

type messageService struct{}

var MessageService = &messageService{}

// GetMessageList 获取聊天记录
// 查询两个用户之间的所有消息，按created_at升序排列
func (m *messageService) GetMessageList(userOneId, userTwoId string) (string, []respond.GetMessageListRespond, int) {
    var messageList []model.Message
    // 查询双向通信记录：(A发给B) OR (B发给A)
    // Order("created_at ASC") 确保消息按时间顺序排列
    if res := dao.GormDB.Where(
        "(send_id = ? AND receive_id = ?) OR (send_id = ? AND receive_id = ?)",
        userOneId, userTwoId, userTwoId, userOneId
    ).Order("created_at ASC").Find(&messageList); res.Error != nil {
        zlog.Error(res.Error.Error())
        return constants.SYSTEM_ERROR, nil, -1
    }
    
    var rspList []respond.GetMessageListRespond
    for _, message := range messageList {
        rspList = append(rspList, respond.GetMessageListRespond{
            SendId:     message.SendId,
            SendName:   message.SendName,
            SendAvatar: message.SendAvatar,
            ReceiveId:  message.ReceiveId,
            Content:    message.Content,
            Url:        message.Url,
            Type:       message.Type,
            FileType:   message.FileType,
            FileName:   message.FileName,
            FileSize:   message.FileSize,
            CreatedAt:  message.CreatedAt.Format("2006-01-02 15:04:05"),
        })
    }
    return "获取聊天记录成功", rspList, 0
}
```

### 1.6 数据库操作说明

**SQL语句解析**：

```sql
SELECT * FROM message 
WHERE (send_id = 'Uxxx1' AND receive_id = 'Uxxx2') 
   OR (send_id = 'Uxxx2' AND receive_id = 'Uxxx1')
ORDER BY created_at ASC;
```

- `WHERE` 条件使用 `OR` 连接两个条件，覆盖双向通信
- `Order("created_at ASC")` 按创建时间升序排列，聊天界面需要按时间顺序展示消息

### 1.7 关键概念详解

#### created_at ASC排序的重要性

聊天消息必须按时间顺序展示，用户才能正常阅读对话内容：

- **ASC（升序）**：旧消息在前，新消息在后，符合聊天阅读习惯
- **DESC（降序）**：新消息在前，旧消息在后，适用于消息列表预览

如果使用 DESC 排序，用户打开聊天界面会先看到最新消息，需要向上滚动才能看到历史消息，这与常见聊天应用的用户体验不符。

## 二、POST /message/getGroupMessageList - 获取群聊消息列表

### 2.1 接口概述

获取群聊中的所有消息记录。群聊消息的接收者是群组（GroupId），所有群成员发送的消息都存储在同一个群组接收记录中。

**核心特点**：
- 只查询 `receive_id = groupId` 的消息（群组是接收方）
- 同样按 `created_at ASC` 升序排序

| 消息范围 | 两用户之间的消息 | 所有群成员的消息 |

## 三、POST /message/uploadAvatar - 上传头像

### 3.1 接口概述

上传用户头像图片。该接口使用 `multipart/form-data` 格式传输文件，与普通的JSON请求不同。

**核心特点**：
- 使用 `c.FormFile()` 获取上传的文件
- 使用 `c.SaveUploadedFile()` 或 `io.Copy()` 保存文件
- 文件存储在配置的头像静态目录中

### 3.2 Request结构体说明

**注意**：UploadAvatar接口不使用BindJSON解析请求，因为请求格式是 `multipart/form-data`。

前端请求格式：
```http
POST /message/uploadAvatar
Content-Type: multipart/form-data

--boundary
Content-Disposition: form-data; name="file"; filename="avatar.jpg"
Content-Type: image/jpeg

[文件二进制数据]
--boundary--
```

### 3.3 Respond结构体定义

上传文件接口不需要复杂的数据响应，只需返回成功/失败状态：

```go
// 成功响应
{
    "code": 200,
    "message": "上传成功"
}

// 失败响应
{
    "code": 500,
    "message": "系统错误"
}
```

### 3.4 Controller层完整代码

```go
// api/v1/message_controller.go

    "kama-chat-server/pkg/zlog"

// UploadAvatar 上传头像
// 注意：不使用BindJSON，使用c.FormFile获取文件

    message, ret := gorm.MessageService.UploadAvatar(c)
    JsonBack(c, message, ret, nil)
```

### 3.5 Service层完整代码

```go
// internal/service/gorm/message_service.go
package gorm

import (
    "fmt"
    "io"
    "os"
    "path/filepath"

    "kama-chat-server/internal/config"
    "kama-chat-server/pkg/constants"
    "kama-chat-server/pkg/zlog"

    "github.com/gin-gonic/gin"
)

// UploadAvatar 上传头像
func (m *messageService) UploadAvatar(c *gin.Context) (string, int) {
    // 解析multipart/form-data请求，限制最大文件大小
    if err := c.Request.ParseMultipartForm(constants.FILE_MAX_SIZE); err != nil {
        zlog.Error(err.Error())
        return constants.SYSTEM_ERROR, -1
    }
    
    mForm := c.Request.MultipartForm
    // 遍历所有上传的文件
    for key := range mForm.File {
        // 使用FormFile获取文件和文件头信息
        file, fileHeader, err := c.Request.FormFile(key)
        if err != nil {
            zlog.Error(err.Error())
            return constants.SYSTEM_ERROR, -1
        }
        defer file.Close()
        
        zlog.Info(fmt.Sprintf("文件名：%s，文件大小：%d", fileHeader.Filename, fileHeader.Size))
        
        // 获取文件扩展名
        ext := filepath.Ext(fileHeader.Filename)
        zlog.Info(ext)
        
        // 构建本地存储路径
        localFileName := config.GetConfig().StaticAvatarPath + "/" + fileHeader.Filename
        
        // 创建本地文件
        out, err := os.Create(localFileName)
        if err != nil {
            zlog.Error(err.Error())
            return constants.SYSTEM_ERROR, -1
        }
        defer out.Close()
        
        // 将上传的文件内容复制到本地文件
        if _, err := io.Copy(out, file); err != nil {
            zlog.Error(err.Error())
            return constants.SYSTEM_ERROR, -1
        }
        
        zlog.Info("完成文件上传")
    }
    return "上传成功", 0
}
```

### 3.6 数据库操作说明

上传头像接口不涉及数据库操作，主要是文件系统操作：

1. 解析multipart请求
2. 获取文件信息
3. 创建本地文件
4. 复制文件内容

用户头像URL更新需要调用 `/user/updateUserInfo` 接口。

### 3.7 关键概念详解

#### multipart/form-data详解

在POST请求中，`enctype`属性规定了表单发送到服务器时的编码方式：

1. **application/x-www-form-urlencoded**：默认编码方式，适合文本传输，不适合大型文件
2. **multipart/form-data**：指定传输数据为二进制类型，适合图片、文件等
3. **text/plain**：纯文本传输，空格转换为"+"加号

#### ParseMultipartForm函数

```go
// ParseMultipartForm解析请求体中的multipart/form-data数据
// maxMemory：最多存储maxMemory字节到内存中，剩余存储到磁盘
func (r *Request) ParseMultipartForm(maxMemory int64) error
```

解析成功后，数据存储在 `MultipartForm` 结构体中：

```go
// MultipartForm结构体
type Form struct {
    Value map[string][]string    // 表单字段值
    File  map[string][]*FileHeader // 上传文件信息
}

// FileHeader文件头结构体
type FileHeader struct {
    Filename string              // 文件名
    Header   textproto.MIMEHeader // MIME头信息
    Size     int64               // 文件大小
}
```

#### FormFile获取文件

```go
// FormFile返回指定key的第一个文件
func (r *Request) FormFile(key string) (multipart.File, *multipart.FileHeader, error)
```

返回的 `multipart.File` 是一个接口，实现了以下方法：

```go
type File interface {
    io.Reader    // 读取文件内容
    io.ReaderAt  // 从指定位置读取
    io.Seeker    // 定位读写位置
    io.Closer    // 关闭文件
}
```

#### Gin框架的简化方式

Gin框架提供了更简洁的文件处理方法：

```go
// 方式1：直接获取文件
file, err := c.FormFile("file")

// 方式2：直接保存文件（更简洁）
err := c.SaveUploadedFile(file, "./static/avatars/"+file.Filename)
```

简化版代码示例：

```go
func UploadAvatar(c *gin.Context) (string, int) {
    file, err := c.FormFile("file")
    if err != nil {
        zlog.Error(err.Error())
        return constants.SYSTEM_ERROR, -1
    }
    
    // 直接保存文件
    savePath := config.GetConfig().StaticAvatarPath + "/" + file.Filename
    if err := c.SaveUploadedFile(file, savePath); err != nil {
        zlog.Error(err.Error())
        return constants.SYSTEM_ERROR, -1
    }
    
    return "上传成功", 0
}
```

上传文件接口本身不操作数据库。文件上传成功后，前端需要：

1. 获取文件URL（通过静态文件服务）
2. 发送消息时携带文件信息，存入Message表

### 4.7 关键概念详解

#### 头像与文件存储分离

```go
// 配置文件
[staticSrcConfig]
staticAvatarPath = "./static/avatars"  // 头像存储路径
staticFilePath = "./static/files"      // 文件存储路径
```

分离存储的好处：
- 便于管理和清理
- 头像文件通常较小且需要长期保存
- 聊天文件可能较大，可以单独设置清理策略

## 五、静态文件路由配置

### 5.1 Gin.Static配置

在HTTP服务器初始化时，需要配置静态文件路由：

```go
// internal/https_server/https_server.go
package httpsserver

import (
    "kama-chat-server/internal/config"

    "github.com/gin-contrib/cors"
    "github.com/gin-gonic/gin"
)

var GE *gin.Engine

func init() {
    GE = gin.Default()
    
    // CORS跨域配置
    corsConfig := cors.DefaultConfig()
    corsConfig.AllowOrigins = []string{"*"}
    corsConfig.AllowMethods = []string{"GET", "PUT", "POST", "DELETE"}
    corsConfig.AllowHeaders = []string{"Orign", "Content-Type"}
    GE.Use(cors.New(corsConfig))
    
    // 静态文件服务配置
    conf := config.GetConfig()
    GE.Static("/static/avatars", conf.StaticSrcConfig.StaticAvatarPath)
    GE.Static("/static/files", conf.StaticSrcConfig.StaticFilePath)
    
    // 注册路由
    registerRoutes()
}
```

### 5.2 静态文件访问原理

`GE.Static(relativePath, root)` 设置静态文件路由：

- `relativePath`：URL路径前缀（如 `/static/avatars`）
- `root`：本地文件目录（如 `./static/avatars`）

**访问示例**：

假设服务器地址为 `http://127.0.0.1:8000`，上传的头像文件名为 `avatar_U123456.jpg`：

```
访问URL: http://127.0.0.1:8000/static/avatars/avatar_U123456.jpg
实际路径: ./static/avatars/avatar_U123456.jpg
```

### 5.3 前端使用静态文件URL

上传头像成功后，前端获取头像URL：

```javascript
// 上传成功后，构建头像URL
const avatarUrl = `http://${serverHost}:${serverPort}/static/avatars/${filename}`;

// 更新用户信息，将avatarUrl存入UserInfo
updateUserInfo({ avatar: avatarUrl });
```

## 十、总结

本章节介绍了四个核心接口的实现：

1. **GetMessageList**：获取私聊消息，使用双向查询条件和 `created_at ASC` 排序
2. **GetGroupMessageList**：获取群聊消息，查询 `receive_id = groupId` 的所有消息
3. **UploadAvatar**：上传头像，使用 `multipart/form-data` 格式，存储到头像目录
4. **UploadFile**：上传文件，同样使用 `multipart/form-data`，存储到文件目录

关键技术点：
- GORM的 `Order()` 方法实现排序
- Gin的 `FormFile()` 和 `SaveUploadedFile()` 处理文件上传
- `ParseMultipartForm()` 解析multipart请求
- `GE.Static()` 配置静态文件服务
- Message.Type枚举区分不同消息类型
